Directed Acyclic Graphs (DAGs)

  • A DAG is a way to model a system of causal interactions using graphs.

  • Nodes represent random variables \(X\), \(Y\),‚Ä¶

  • Arrows or directed edges represent “from \(\rightarrow\) to” causal effects: \(Z\rightarrow X\) reads “\(Z\) causes \(X\)”

  • A path is a any sequence of edges connecting two nodes: \(Z\rightarrow X \rightarrow M \leftarrow Y\) is a path from \(Z\) to \(Y\).
  • In a direct path arrows point to the same direction: \(Z\rightarrow X \rightarrow M\)

Confounder DAG

  • \(X\) is a common cause of \(D\) and \(Y\).

  • Conditioning on \(X\) removes dependency between \(D\) and \(Y\) through \(X\).

  • In DAG terminology, controlling for X “closes the backdoor path” between \(D\) and \(Y\), and leaves open the direct path.

  • This notion of closing the backdoor path is related to the notion of omitted variable bias.

DAGs and SEM

  • DAGs can be considered as non-parametric structural equation models (SEM)

  • The previous single-confounder DAG can be represented by a set of three equations:

\[ \small \begin{array}{l} X\leftarrow f_X\left(u_{X}\right) \\ D\leftarrow f_D\left(X, u_D\right)\\ Y\leftarrow f_Y(D,X,u_Y) \end{array} \] - \(f_{i}\) denote the causal mechanisms in the model (in general, non-linear)

  • \(u_X,u_D\), and \(u_Y\) denote independent background factors that the we chooses not to include in the analysis.

  • Assignment operator \((\leftarrow)\) captures asymmetry of causal relationships.

A mediator

  • \(D\) causes \(M\) causes \(Y\).

  • \(M\) mediates the causal effect of \(D\) on \(Y\)

  • Conditioning on \(M\) removes dependency between \(D\) and \(Y\)

  • In essence, we have closed a direct path (the only direct path between \(D\) and \(Y\).

A Collider

  • \(D\) are \(Y\) are independent.

  • \(D\) and \(Y\) jointly cause \(C\).

  • Conditioning on \(C\) creates dependency between \(D\) and \(Y\)!

DAG analysis with dagitty

  • Shortest path, path coefficients identifiable by regressions or by IV, etc.
g1 <- dagitty( "dag {
    W1 -> Z1 -> X -> Y
    Z1 <- V -> Z2
    W2 -> Z2 -> Y
    X <-> W1 <-> W2 <-> Y
}")
g1 %>% ggdag()
for( n in names(g1) ){
    for( m in children(g1,n) ){
        a <- adjustmentSets( g1, n, m, effect="direct" )
        if( length(a) > 0 ){
            cat("The coefficient on ",n,"->",m,
                " is identifiable controlling for:\n",sep="")
            print( a, prefix=" * " )
        }
    }
}

nx.is_directed_acyclic_graph(graph)

Summarizing DAGs

  • “Confounders” and "colliders are always context-specific

    • Your dataset will not come with a codebook flagging your variables as one or another

  • You do not know in general if you have an omitted variable bias problem

  • All identification must be guided by theory, experience, observation, common sense and knowledge of institutions

  • DAGs absorb that information and can be then used to write out the explicit identifying model
